## 📍 현재 상태 (2026-06-21 — 렌탈 튀는 현상 수정)

> admin.html 폴링 → MutationObserver 교체로 새로고침 시 렌탈 튀는 현상 해결.
> 커밋 `96fd14c`. working tree 깨끗.

### ✅ 새로고침 시 렌탈 패널 튀는 현상 — 완전 해결

- **원인**: admin.js `init()` 내 async 2차 `applyMenuFromHash()`(line 3286)가 `banner.hidden=true` / `products.hidden=false`로 덮어씀
- **해결**: `[data-panel="banner"]`에 MutationObserver → `hidden` 속성이 true가 되는 즉시 `applyPanel()` 호출로 복원
  - 복원 후 `observer.disconnect()` → 이후 수동 탭 전환에 간섭 없음
  - 10초 안전망 setTimeout으로 MO 해제
  - 4초 한계 있던 폴링 방식 완전 제거
- **검증**: 5초 대기 후 DOM 상태 확인 — `dataShop:phone`, `bannerHidden:false`, `productsHidden:true` ✓

### ✅ 본부전용 패널 3개 (admin.html 인라인 IIFE)
- 사이드바 그룹 `본부 전용` → 모델관리 / 요금제관리 / 공통지원금 (data-shop-menu 라우팅)
- 각 패널: SKT/KT/LG U+ 탭 + CRUD (모달), 공통지원금 = 인라인 셀 편집(blur 저장)
- Supabase 테이블: `mobileshop_models` / `mobileshop_plans` / `mobileshop_support`
- 클라이언트: `window.sb` (렌탈 admin은 `window.sb` 사용 — `window.supabaseClient` 아님)
- 모델 엑셀 내보내기/가져오기(xlsx) 지원

### ✅ 관리자 새로고침 상태유지
- localStorage `adm_shop` / `adm_menu` 저장 → 새로고침 시 탭+패널 복원
- 깜빡임 제거: 인라인 `<script>`로 `data-preload-shop` 세팅 → CSS로 렌탈탭 색상 숨김

### 🧠 다음 세션 알아둘 것 / 함정
- admin.js는 IIFE 내부 함수 — `applyMenuFromHash` 외부에서 오버라이드 불가
- admin.js `applyMenuFromHash` 호출 위치: `init()` line 3220(동기), line 3286(비동기) + hashchange 이벤트
- `parseHash()`: 빈 해시 → 기본값 `'products'` (렌탈 상품관리 패널 오픈)
- `body[data-shop]` 전환 = `setShop()` 함수, 렌탈 전환 시 hashchange 발화
- MO 코드 위치: admin.html 하단 shop controller IIFE 내 DOMContentLoaded 콜백

- 원인: `persistOne`/`saveOverrides`가 **구 `admin_overrides` 테이블에 직접 write** → woozoo-shop엔 그 테이블이 없어 실패.

### ✅ 본부 전용 메뉴 노출 (단일샵 운영자 = 오너)
- `판매점마진설정`·`제휴카드 관리`·`아이콘 시안`을 **본부 전용 그룹**으로 노출(`applyRoleVisibility`에서 항상 표시, `_isSuper`는 안 건드림 → 상담스코프·수수료차감 부수효과 회피). **사이트분양은 단일샵 미사용이라 계속 숨김.** 판매점마진은 단일샵 슬러그라 평면(flat) 모드 렌더(UI만, 저장 stub).

### ✅ 렌탈 페이지 PC 레이아웃 (= 단독 skmagic-dealer 처럼)
- `rental.html`이 `!important`로 모바일 레이아웃 강제하던 걸 **`@media(max-width:767px)`로 가둠** → 데스크탑 폭에선 `style.css`의 PC 레이아웃(컨테이너 1200·product-grid 4열·2단 히어로) 적용. + 데스크탑 언프레임(풀폭)·헤더 가로배열(합본 메뉴 유지). 임베드(렌탈 탭)·단독 `/rental` 둘 다 적용.

### ✅ PC 3탭 헤더 동작/구조 통일
- 스크롤해도 **대메뉴 안 숨고**(데스크탑 compact 비활성), **소메뉴가 헤더 바로 아래 고정**. 렌탈 전용 compact 트릭(헤더 clip·iframe 끌어올림) 데스크탑 비활성.
- **렌탈 소메뉴(브랜드 nav)를 부모(index.html `#rental-sub-nav`)로 이전** → 3탭 모두 `부모 .head + 부모 .sub-nav + 콘텐츠` 동일 구조. iframe 자체 `.msr-sub`/`.filter-bar`는 임베드 시 숨김.
- 휴대폰/인터넷은 소메뉴를 배너 위(`order:-1`)로 올려 대메뉴와 붙임.
- 렌탈 3단 sticky 헤더(상품군 pill 탭) 만들었다가 → **사용자 요청으로 3단 제거, 2단(메인네비+브랜드)으로 환원**. 카테고리 전환은 iframe 자체 `.filter-bar` 복원으로.

### ✅ 새로고침 상태 유지 + 깜빡임 제거
- 탭 전환 → `history.replaceState ?cat=` 기록, 로드 시 복원. 렌탈 하위경로는 `app.js route()`가 `msr-route` 메시지로 `?cls=/?id=` 전달 → 부모 URL 미러 → `mountRentalFrame`이 그 경로로 iframe 마운트.
- **FOUC 제거**: `/scroll` 직후 인라인 스크립트가 첫 페인트 전 `?cat=` 패널위치·활성탭 즉시 적용(휴대폰 잠깐 보였다 넘어가던 것 제거).

### ✅ 렌탈 배너 — 버그/크기/PC·모바일 SET
- 저장해도 안 뜨던 버그: `#hero-slider` 제거됐는데 JS가 그걸 찾아 early-return → **`.banner-wide` 채우게 수정**.
- 크기: 단독 `.hero-slider`와 동일 — **max-width 1440 · aspect 16/5 · 중앙정렬**(넓은 모니터 풀스크린 넘침 해소). **노출 크기 = 최대 1440×450(데스크탑) / ~375×117(모바일), 16:5 cover.**
- **PC·모바일 SET**: 관리자 `+ 배너 추가`로 빈 SET 행 → PC/모바일 슬롯 각각 업로드. 데이터 `items[].image(PC)+imageMobile`. 손님 페이지가 뷰포트(≤767=모바일)에 맞춰 선택(없는 쪽 폴백). 배너 URL `{pc,mobile}` JSON 캐시 → 새로고침 시 첫 페인트에 즉시 적용(샘플 잔재 제거). 관리자 미리보기 카드형으로 확대(PC 큰 미리보기/모바일 좁게).

### 🧠 다음 세션 알아둘 것 / 함정
- **렌탈 = iframe**(`rental.html`, src `/rental?embed=1`). 부모 `index.html`이 1단 헤더(.head) 제공, 임베드 시 iframe의 `.msr-head/.msr-sub/.filter-bar` 숨김. 라우팅은 iframe 내부 `<a href="/rental?cls=">` 클릭 가로채기(app.js).
- **정책표 매칭** `comHomeMap` = PRODUCTS_DB raw + VISIBLE_CATS 필터(PSG 고유이름 위해 raw 사용, _siblings 쓰면 안 됨).
- **오버라이드 저장** = `skmUpsertOverride`(`mobileshop_rental_overrides`, store_id 없음, onConflict goods_id). `featured_rank` 컬럼 추가됨.
- **프리뷰 도구 함정**: 렌탈 iframe의 외부이미지(supabase) 로딩 시 `preview_screenshot` 타임아웃 잦음 → 좌표/상태는 `preview_eval`로 검증. 캐러셀 `left:-200%`가 안 밀려 보이면 트랜지션 stuck(transition:none로 확인).
- 미해결/후보: 모바일 배너를 더 도톰한 비율(16:5 아닌)로 보이게 할지 결정 보류. 휴대폰/인터넷 패널은 아직 '모바일 확대' 수준(렌탈처럼 진짜 PC 레이아웃 미적용).

- 드래그 중(`input` 이벤트): CSS 변수 1개만 postMessage → iframe이 setProperty 1번. 60Hz 부담 0
- 드래그 떼면(`change` 이벤트): `pushPreview()` 1회로 workingData ↔ 폼 ↔ iframe 동기화
- **5~10개까지는 직접 핸들러로 OK, 더 가면 레지스트리 패턴(`SURGICAL` 배열에 정의 → 자동 와이어링)**

**드물지만 빠지는 함정**
- 공통 `chip-emph` 클래스 재사용했더니 CTA(체크박스 없음)에서 `.checked` null 접근으로 load 전체가 죽었음 → 항상 **null 체크**
- `drop-shadow(0 12px 0 rgba)`는 blur=0이어도 sharp shadow가 12px 아래 찍힘 → 값을 offset·blur·opacity 모두에 곱해서 0 → 완전 투명
- CTA `<b>` 그라데이션 텍스트는 `background-clip:text;color:transparent` → 부모 `.bw-cta`에 인라인 color 줘도 안전 (`<b>`는 영향 X)
- 옛 `'left'/'center'/'right'` 데이터는 자동으로 0/50/100 변환 (`normalizeAlign`) — 데이터 마이그레이션 없이 호환

**z-index 레이어**
- `.banner-wide::before` (배경 그라데이션) = 0
- `.bw-phone` (이미지/SVG) = 1
- `.bw-tags`, `.bw-title`, `.bw-cta-row` = 2 → 이미지가 텍스트 영역 침범해도 글씨 안 가려짐

**데이터 모델 (이번 세션 추가분)**

```js
  tags: [{ text, hot, bg, fg }, ...],
  eyebrow, eyebrowColor, eyebrowAlign,

  ctaText, ctaTextColor, ctaAlign,
  ctaHighlight, ctaHlStart, ctaHlEnd,
  imageUrl (data URL),
  imageRotate, imageScale, imageOffsetX, imageOffsetY,
  imageShadow, imageGlow,
  chipAlign, background
}
```

### ⚠️ 미해결 이슈 — head ↔ sub-nav 글래스 mismatch
sub-nav가 head에 stuck될 때 두 영역의 `backdrop-filter`가 서로 다른 픽셀을 샘플링해 색·블러 어긋남 + 1px 이음새 보임.

시도한 3가지 모두 trade-off 있어 보류:
- **(a) DOM 이동 (sub-anchor + sub-slot)**: 느린 스크롤 완벽, 빠른 스크롤 시 위치 점프
- **(b) `head::after`로 글래스 영역 확장**: pseudo-element + 부모 backdrop-filter 중첩 호환성 이슈로 sub-nav 영역이 그냥 투명하게만 보임 (backdrop-filter가 안 먹음)
- **(c) head `padding-bottom` 트릭 (margin-bottom 음수로 in-flow 보존)**: (b)와 동일 결과

**현재 상태**: sticky 유지 + stuck 시 head의 `border-bottom-color: transparent`만 (미세 mismatch 잔존하지만 이음새는 사라짐)

### 핵심 패턴 (변경/유지)
- **가로 carousel**: `.panels { width:400%; display:flex }`, `.panel { width:25% }`, JS `panelsEl.style.left = -idx*100 + '%'`
- **`transform` 안 씀** — `transform`이 자식 sticky를 깸. `left` transition으로 carousel 흉내 (살짝 덜 부드러울 수 있으나 sticky 작동)

- **sub-nav sticky** (top:77px, z-index:32) — panel 안 배너 다음 위치 유지
- **카테고리 전환 시 스크롤 0**: `scrollEl.scrollTop=0` + rAF 보험 + `window.scrollTo(0,0)` 3중
